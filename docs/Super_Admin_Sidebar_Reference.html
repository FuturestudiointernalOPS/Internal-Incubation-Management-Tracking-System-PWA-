Super Admin Sidebar and Module Reference

ImpactOS Future Studio — Complete navigation, purpose, and testing guide for the Super Admin role. Based on commit 1af98fe.

1. Main Operations

Menu Type Route What It Does How to Test
DASHBOARD Direct Link /admin Command Center. Shows: Unified Calendar (month/week/day views, color-coded events: blue=task, green=program, amber=session, purple=deliverable), Summary Cards (My Programs, My Projects, Open Tasks, Active Blockers, Overdue, Critical), Assigned To Me with Accept/Decline/Complete workflow, Attention Required (overdue tasks, critical blockers, due today), Recent Activity feed, Quick Access panels for Programs/Projects/Tasks/Blockers (max 5 records, View All buttons). Log in as super_admin. Verify calendar renders events from all sources. Click each summary card — they navigate to correct module. Verify Assigned To Me shows pending tasks. Click Accept — status changes. Click a calendar event — detail drawer opens with date, source, type, status. Toggle month/week/day views.

2. PROGRAMS (Dropdown — Briefcase icon)

Sub-Menu Route What It Does How to Test
ALL PROGRAMS /admin/programs Lists all programs. Columns: Name, Status, Assigned PM, Participants, Deliverables, Completion Index, Created Date. Search/filter by name. Each row clickable to program detail page. Verify table loads all programs. Click a row — navigates to /admin/programs/{id}. Use search bar to filter. Check completion index matches known data.
CREATE PROGRAM /admin/programs/new Form: Name (required), Description, Start Date, End Date, Status dropdown (Active/Inactive/Completed), Assign PM (dropdown of available PMs). Fill all fields. Submit. Verify new program appears in ALL PROGRAMS. Verify assigned PM sees it on their dashboard. Verify program detail page loads correctly.
PROGRESS /admin/progress Program performance dashboard. Visual charts and metrics: completion rates, participant engagement, deliverable tracking across all programs. Verify charts render with real program data. Compare completion percentages against known program stats. Hover for tooltips.
PROGRAM REPORTS /admin/reports/responses All submitted PM weekly report responses. Historical archive. Shows: Program name, PM name, week number, year, submission date, full report content. Verify reports load. Filter by program name or date range. Click to expand full response. Verify timestamps are correct.

3. PROJECTS (Dropdown — Rocket icon)

Sub-Menu Route What It Does How to Test
ALL PROJECTS /admin/projects Lists all internal projects. Columns: Name, Status (Active/Paused/Completed), Owner, Tasks (total), Blockers (active), Completion Rate (%). Click row to open project detail with 7 tabs: Overview, Tasks, Blockers, Team, Update, Approvals, Timeline. Verify table loads all projects. Click a project — detail page shows 7 tabs. Check each tab renders: tasks list, blockers list, team members, weekly updates, approval requests. Verify completion rate matches data.
MY PROJECTS /staff Project execution workspace. Shows: Quick Actions (Standup, Retro, Projects), Assigned To Me with Accept/Decline/Complete, My Projects table (Project name, Role: Owner/Collaborator, Tasks count, Blockers count, Post Update button for owners), Project Blockers list with Resolve button. No calendar or dashboard cards. Verify only YOUR projects appear. Owned = Rocket icon + Owner badge. Collaborator = Users icon + Collaborator badge. Click Post Update — weekly update modal opens (accomplishments, focus, blockers, next steps, overall status). Submit — verify it saves. Click Resolve on a blocker — blocker resolves, list refreshes. Accept/Decline an assigned task.

4. WORK (Dropdown — ListTodo icon)

Sub-Menu Route What It Does How to Test
WORK HUB /admin/work Centralized hub for all work items: tasks, projects, categories, blockers in one unified view. Quick-create tasks and blockers. Verify page loads all work items. Create a new task. Create a new blocker. Verify they appear in respective lists.
TASKS /admin/tasks Full task management. Create/edit/delete tasks. Status workflow: pending to in_progress to completed (blocked if blockers active). Assignment to any user (they receive notification). Dates: start_date, end_date. Priority levels. Filter by status, user, project, date range. Create a task with title, description, assignee, dates. Change status through workflow. Add a blocker — status changes to blocked. Assign to another user — they see it under Assigned To Me. Filter by status blocked — only blocked tasks show.
BLOCKERS /admin/blockers All blockers across the system. Filter by status (active/resolved), severity (critical/high/medium/low). Sort by severity. Resolve any blocker (admin override) — automatically reverts task to in_progress if no other blockers remain. Create a blocker on a task. Verify it appears with correct severity badge. Filter by active. Click Resolve — blocker resolved, task unblocked. Filter by critical. Verify sorting works (critical first).

5. REPORTS (Dropdown — BarChart3 icon)

Sub-Menu Route What It Does How to Test
INTERNAL REPORTS /admin/op-reports All submitted standups (Monday) and retros (Friday). Filter by user, report type, week/year. Per-user consistency metrics: reports submitted, blocker rate, last report date. Submit a standup from a staff account. Verify it appears here. Filter by user. Check consistency metrics (consistent = 4+ reports, at-risk = 1-3, inactive = 0).

6. COMMUNICATIONS (Dropdown — Send icon)

Sub-Menu Route What It Does How to Test
CAMPAIGNS /admin/communications/campaigns Bulk communication tool. Create messaging campaigns targeted to programs, groups, or specific contacts. Track delivery status. Create a test campaign. Select recipients. Write message. Send. Verify delivery status updates.
FORMS /admin/communications/forms Form builder for intake, feedback, surveys. Create forms with various field types. Publish for participant/staff access. View submissions. Create a form with text, select, date fields. Publish it. Switch to participant account — verify form is accessible. Submit response. Switch back to admin — verify submission appears.
CONTACTS /admin/communications/contacts Full user directory. All contacts across all roles. Search by name/email. Filter by role, status (active/pending/inactive). Approve pending user registrations. Verify all contacts load. Search by email. Filter by role teacher. Find a pending user — Approve — verify status changes to active.
PENDING USERS /admin/pending-users Users awaiting account approval. Quick Approve/Reject actions. Shows name, email, registration date, requested role. Register a new user. Verify they appear here. Approve — user becomes active. Verify they can log in. Reject — user is removed.
BULK UPLOAD /admin/bulk-upload Mass import users/contacts via CSV. Download template, populate, upload. Validates and imports in batch. Download CSV template. Add 3-5 test users. Upload file. Verify users appear in Contacts with correct roles.

7. Standalone Links

Menu Type Route What It Does How to Test
KNOWLEDGE BASE Direct Link /admin/knowledge Internal documentation repository. Create, edit, organize articles. SOPs, guides, reference materials. Create a new article. Verify it renders. Edit — verify changes save. Delete — verify removed from list.
INTELLIGENCE Direct Link /admin/intelligence Analytics and insights hub. Historical trends, engagement metrics, program performance over time. Verify charts load with real data. Compare metrics against known program stats. Check date range filters work.

8. User Protocol (Footer)

Menu Type Route What It Does
PROFILE Direct Link /admin/profile View and edit your profile: name, email, role display, notification preferences.
LOGOUT Action Button — Destroys the server session, clears local storage, redirects to /login.

9. Suggested End-to-End Test Flow

  1. Login as super_admin — verify DASHBOARD loads with calendar and summary cards
  2. Create a Program via CREATE PROGRAM — verify it appears in ALL PROGRAMS
  3. Create a Project from ALL PROJECTS — verify detail page with all 7 tabs
  4. Create a Task — assign to another user — verify they see it under Assigned To Me
  5. Add a Blocker to the task — verify status changes to "blocked"
  6. Check MY PROJECTS (/staff) — verify your projects, tasks, blockers appear
  7. Submit a Weekly Update — click Post Update on a project — fill form — submit
  8. Resolve a Blocker — from MY PROJECTS or DASHBOARD — verify task unblocks
  9. Accept an Assignment — from Assigned To Me — click Accept — status changes to in_progress
  10. Submit a Standup — from Quick Actions — fill report — verify it appears in INTERNAL REPORTS
  11. Approve a Pending User — from PENDING USERS or CONTACTS — verify they can log in
  12. Send a Campaign — from COMMUNICATIONS — select recipients — send — verify delivery
